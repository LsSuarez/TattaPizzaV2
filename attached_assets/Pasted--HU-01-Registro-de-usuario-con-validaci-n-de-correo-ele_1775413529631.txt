## HU-01: Registro de usuario con validación de correo electrónico

*Declaración:*

* *YO COMO* cliente nuevo de la pizzería
* *QUIERO* registrarme en la plataforma web proporcionando mis datos personales y validando mi correo electrónico
* *CON EL OBJETIVO* de tener una cuenta segura que me permita realizar pedidos, guardar mis preferencias y hacer seguimiento de mis órdenes

*Criterios de Aceptación:*

1. *Dado que* el usuario accede a la página de registro y completa todos los campos obligatorios (nombre, correo electrónico, contraseña, teléfono, dirección)  
   *Cuando* presiona el botón "Registrarse"  
   *Entonces* el sistema envía un correo electrónico de verificación con un enlace único de activación y muestra un mensaje indicando que debe revisar su bandeja de entrada

2. *Dado que* el usuario recibe el correo de verificación y hace clic en el enlace de activación  
   *Cuando* el enlace es válido y no ha expirado (máximo 24 horas)  
   *Entonces* el sistema activa la cuenta, redirige al usuario a la página de inicio de sesión y muestra un mensaje de confirmación de cuenta activada

3. *Dado que* el usuario intenta registrarse con un correo electrónico que ya existe en el sistema  
   *Cuando* presiona el botón "Registrarse"  
   *Entonces* el sistema muestra un mensaje de error indicando que el correo ya está registrado y sugiere iniciar sesión o recuperar contraseña

*Prioridad:* P0  
*Estimación:* 5 puntos

1. *Dado que* el usuario tiene una cuenta activa y verificada

   *Cuando* intenta iniciar sesión nuevamente  
   *Entonces* el sistema bloquea temporalmente la cuenta por 15 minutos, envía una notificación al correo del usuario y muestra un mensaje indicando el tiempo de espera

*Prioridad:* P0  
*Estimación:* 3 puntos

## HU-09: Procesamiento automático de órdenes con Edge Functions

*Declaración:*

* *YO COMO* administrador de la pizzería
* *QUIERO* que las órdenes confirmadas y pagadas se procesen automáticamente mediante Edge Functions de Atoms Cloud
* *CON EL OBJETIVO* de eliminar procesos manuales, reducir errores y agilizar la comunicación entre el sistema, la cocina y el cliente sin costos adicionales

*Criterios de Aceptación:*

1. *Dado que* un cliente completa exitosamente el pago de su pedido (o selecciona pago contra entrega)  
   *Cuando* el sistema recibe la confirmación  
   *Entonces* una Edge Function se ejecuta automáticamente y: registra la orden en la base de datos de Atoms Cloud, actualiza el panel del pizzero en tiempo real, envía un correo de confirmación al cliente y actualiza el estado del pedido a "En espera"

2. *Dado que* el pizzero cambia el estado de una orden en su panel  
   *Cuando* el estado cambia de "En espera" a "En producción" o de "En producción" a "Listo para envío"  
   *Entonces* una Edge Function envía automáticamente un correo electrónico al cliente informando el cambio de estado y, en caso de "Listo para envío", notifica al motorizado asignado con los detalles de entrega

3. *Dado que* ocurre un error en algún paso del procesamiento automático  
   *Cuando* el sistema detecta el fallo  
   *Entonces* la Edge Function registra el error en un log de la base de datos, envía una alerta al administrador por correo electrónico y reintenta la operación hasta 3 veces antes de escalar el problema

*Prioridad:* P0  
*Estimación:* 8 puntos

   *Entonces* el sistema agrega automáticamente todos los productos de ese pedido al carrito actual con las mismas personalizaciones, verifica la disponibilidad y precios actuales, y notifica si algún producto ya no está disponible o cambió de precio

3. *Dado que* el usuario desea ver el detalle de un pedido específico  
   *Cuando* hace clic en un pedido del historial  
   *Entonces* el sistema muestra el detalle completo incluyendo: productos con personalizaciones, dirección de entrega utilizada, método de pago, tiempos de cada etapa (pedido, preparación, envío, entrega) y calificación otorgada

## HU-18: Sistema de calificación y reseñas

*Declaración:*

* *YO COMO* cliente que ha recibido su pedido
* *QUIERO* calificar la experiencia de mi pedido (calidad de la pizza, tiempo de entrega, servicio general) y dejar una reseña
* *CON EL OBJETIVO* de compartir mi opinión, ayudar a otros clientes en su decisión y contribuir a la mejora del servicio de la pizzería

*Criterios de Aceptación:*

1. *Dado que* el pedido del cliente ha sido marcado como "Entregado"  
   *Cuando* el cliente accede a la sección "Mis Pedidos" y selecciona el pedido entregado  
   *Entonces* el sistema muestra un formulario de calificación con: estrellas del 1 al 5 para calidad del producto, tiempo de entrega y servicio general, un campo de texto para comentarios opcionales y el botón "Enviar calificación"

2. *Dado que* el cliente completa la calificación con estrellas y opcionalmente un comentario  
   *Cuando* presiona "Enviar calificación"  
   *Entonces* el sistema registra la calificación en la base de datos, actualiza el promedio de calificación de la pizzería, muestra un mensaje de agradecimiento y, si la calificación es menor a 3 estrellas, una Edge Function envía un correo al administrador para seguimiento

3. *Dado que* un cliente potencial navega por el menú  
   *Cuando* visualiza un producto específico  
   *Entonces* el sistema muestra el promedio de calificación del producto, el número total de reseñas y las últimas 5 reseñas con nombre del cliente (parcialmente oculto), calificación y comentario
